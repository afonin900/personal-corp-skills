# Idea Skill

Часть фреймворка Personal Corp.

## Что это

Скилл быстрого захвата одной озвученной идеи с полным провенансом. Работает по принципу **одна папка = одна идея**: каждая идея получает собственный подкаталог в твоём репозитории идей с файлами сути, источников, рынка и следующих шагов.

Перед созданием новой папки скилл проверяет `INDEX.md` на дубли — семантически, не строково. Если идея уже есть, добавляет новую строку в `sources.md` существующей папки вместо создания дубля.

Опционально: зеркалит каждую идею как GitHub Issue с лейблом `idea` и добавляет её на доску Project (настраивается в конфиге).

## Зачем нужно

Без такого скилла идеи теряются:

- Озвучена в разговоре → не записана → забыта.
- Записана повторно → дубль в каталоге → непонятно сколько раз идея всплывала.
- Зафиксирована без источника → через месяц неизвестно, откуда пришла и кто озвучил.
- Записана в issue/заметке без папки → нет места для sources, market, next-steps → идея не развивается.

Скилл закрывает все четыре дыры: дедуп через INDEX до создания, провенанс на каждой идее, структурированные файлы для роста идеи, опциональный срез на доске.

## Когда применять

Скилл для **одной конкретной идеи**, которую ты сейчас озвучиваешь:

- «Запиши идею: ...»
- «У меня идея — ...»
- «Зафиксируй: ...»
- `/idea`
- `idea: {формулировка}`

**Не применять для:**

- Добычи нескольких идей из транскрипта встречи или длинного треда — это задача отдельного extraction-скилла.
- Рабочих задач с дедлайном и исполнителем — для них нужен `manager` / GitHub Issues.

## Что получишь

После вызова скилл возвращает короткий отчёт:

- Путь созданной папки (`ideas/YYYY-MM-DD-{slug}/`)
- Номер issue-зеркала (если зеркало включено в конфиге)
- Ссылка на доску Project (если настроена)

Структура каждой папки идеи:

```
ideas/YYYY-MM-DD-{slug}/
  idea.md        — суть, сырая формулировка, why (статус: seedling → exploring → building → live → parked → dropped)
  sources.md     — таблица: дата / кто / контекст / ссылка на источник
  market.md      — аналоги, потенциал, YC RFS совпадения (опционально, заполняется позже)
  next-steps.md  — что делать дальше: landing? brainstorm? закрыть?
```

`INDEX.md` в корне репозитория перегенерируется автоматически после каждой записи.

## Установка

```bash
cp -r skills/idea ~/.claude/skills/
```

После этого скилл доступен в Claude Code.

## Настройка

Добавь в `CLAUDE.md` твоего проекта секцию `## Idea Config`:

| Ключ | Назначение | Пример |
|------|------------|--------|
| `ideas_repo` | Абсолютный путь к твоему репозиторию идей | `~/Documents/GitHub/my-ideas` |
| `default_author` | Значение `first_voiced_by` по умолчанию | `Me` |
| `owner` | Твой GitHub handle для issue/project команд (только если `mirror_enabled: true`) | `your-github-handle` |
| `ideas_repo_name` | Имя GitHub-репозитория для issue-зеркала (только если `mirror_enabled: true`) | `ideas` |
| `mirror_enabled` | Создавать ли issue + Project зеркало (`true` / `false`) | `false` |
| `project_number` | Номер Project на GitHub (только если `mirror_enabled: true`) | `7` |

Зеркало всегда использует лейбл `idea` (зашит в скилл). Минимальная рабочая конфигурация — только `ideas_repo` и `default_author`. Без `mirror_enabled: true` issue-зеркало и Project-шаги пропускаются.

Без конфига скилл переспросит путь к репозиторию при первом запуске.

## Как запустить

**Быстрая фиксация идеи:**

> «Запиши идею: подкаст с разбором агентных стеков, один эпизод = один реальный кейс»

или

> `/idea`

или

> «idea: ежемесячный open-source дайджест для комьюнити»

Скилл:

1. Определяет реальную дату (`date`; в 00:00–04:00 «сегодня» = прошедший день).
2. Читает `INDEX.md` — семантический поиск дублей. Если идея уже есть → добавляет строку в `sources.md` существующей папки, не создаёт новую.
3. Если идея новая → создаёт папку из шаблона, заполняет `idea.md` (статус `seedling`, сырая формулировка дословно) и `sources.md` (строка провенанса).
4. Перегенерирует `INDEX.md`.
5. Если `mirror_enabled: true` — создаёт issue (`idea: {title}`, лейбл `idea`), тело = ссылка на папку + суть + статус + провенанс, и добавляет issue на доску Project (`project_number` из конфига).
6. Коммитит и пушит в репозиторий идей (`feat: idea {slug}`).
7. Возвращает отчёт одной строкой на пункт.

**Один уточняющий вопрос** — только если из формулировки нельзя понять суть (что это / для кого). Механику, цены, стек, риски не спрашивает: фиксирует сырьё как есть, пробелы записывает в `next-steps.md`.

**Форматы провенанса** (`sources.md`):

| Тип источника | Строка в таблице |
|---|---|
| Прямой вызов скилла | `прямой вызов /idea, сессия Claude Code, {дата}` |
| Заметка из встречи | `заметка из встречи — {кто/где}, {дата}` |
| CRM / сделка | `CRM-карточка — {контакт}, {дата}` |
| DM / переписка | `DM — {канал}, {дата}` |
| Голосовой черновик | `voice draft, {дата}` |
| Чтение / статья | `{URL}, {дата}` |
